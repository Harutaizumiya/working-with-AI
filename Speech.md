# Working with AI — 演讲稿（约 1.5h）

---

# 开场（5 min）

大家好，今天分享的主题是：

**Working with AI**

这不是最佳实践，也不是标准答案，而只是我个人在深度使用 AI 之后的一些经验和观察。

AI 的进化速度非常快，所以任何所谓的方法论，都可能在几个月后就过时。  
我们能做的不是寻找“正确答案”，而是学会与 AI 一起探索。 🚀

今天的内容主要围绕三个问题：

- AI 编程正在如何演进
- 我是如何与 AI 协作工作的
- AI 如何改变开发者的思维方式

---

# 第一部分：从 Tab 到 Agent（8 min）

过去几年，我们经历了一个非常明显的变化：

从 **Tab 补全 → 到 Agent 自动执行**。

最早的时候，我们使用的是行内补全，比如：

- 自动补全函数名
- 自动补全一行代码
- 简单上下文推断

这类工具的特点是：

你仍然在写代码，AI 只是帮你减少键盘输入。

然后进入第二阶段：

**多光标、多位置补全**

AI 不只是补全一行，而是：

- 同时修改多个地方
- 推断结构化代码
- 重构局部逻辑

但真正的变化来自第三阶段：

## Agent

这个阶段的核心变化是：

你不再告诉 AI “写什么代码”  
而是告诉 AI “我要实现什么目标”

AI 开始：

- 理解项目结构
- 修改多个文件
- 运行测试
- 修复错误
- 循环迭代

换句话说：

**你给出目的地，AI 开车 🚗**

这就是范式变化。

# 第二部分：AI 编程自动驾驶等级（10 min）

我们可以把 AI 编程能力类比成自动驾驶等级：

- L0 — 完全手写代码
- L1 — 行内补全
- L2 — 代码片段生成 + IDE chat
- L3 — Coding Agent，人类 review
- L4 — Agent + AI review
- L5 — 完全自主 AI 团队

目前大多数团队处在：

**L2 → L3 之间**

而真正的瓶颈从 L3 开始出现：

**code review**

因为：

- AI 写代码已经很快
- 但人类不敢完全信任

所以 review 成为瓶颈。

换句话说：

**AI 的速度 > 人类理解速度**

未来的关键问题是：

**如何建立信任。**

# 第三部分：工具选择（5 min）

我个人使用的工具组合比较简单：

- 写代码：Coding Agent
- 创意工作：多模型
- Research：通用 AI
- 自动化：脚本 / 浏览器工具

核心原则只有一个：

**工具不是重点，workflow 才是重点**

很多人花时间比较模型：

- 哪个更强？
- 哪个更快？

但实际生产力差异来自：

**你如何组织 AI 工作流程。**

# 第四部分：Skills 的思考（8 min）

很多人喜欢给 AI 写 skill。

但我对 skill 的看法比较谨慎。

可以用一个比喻：

传统编程像棋谱  
AI 更像 AlphaZero

棋谱是人总结的经验  
AlphaZero 是自己发现策略

当模型足够强时：

**固定 skill 反而可能限制 AI。**

Skill 有用的场景：

- 特殊工具
- 冷门领域
- 非主流技术栈

Skill 不太有用：

- 主流语言
- 主流框架
- 模型已经很熟的领域

核心观点：

**skill 是索引，不是规则**

# 第五部分：CLI vs MCP（6 min）

两种工具调用方式：

- CLI
- MCP

CLI 的特点：

- 快
- 可组合
- 易调试

MCP 的特点：

- 可发现
- 标准化
- 跨客户端

最佳实践不是二选一，而是：

**CLI 做底层，MCP 做接口层**

# 第六部分：Workflow（10 min）

我的 AI 开发 workflow：

关键点：

**不要直接让 AI 写代码。**

而是：

- 先研究
- 再规划
- 再实现

Agent 内部也有一个循环：

所以：

- 人类负责外循环
- AI 负责内循环

---

# 第七部分：Research 双模式（6 min）

Research 有两种模式：

### Context-free

- 不带偏见
- 从第一性原理思考

### Context-rich

- 结合项目上下文
- 优化现有架构

最佳策略：

**先 context-free，再 context-rich**

---

# 第八部分：Prompt 经验（10 min）

我的 prompt 原则：

- 基于第一性原理
- 从根源解决问题
- 保持中立态度
- 先理解再修改
- 相信 AI 的能力

一个重要变化：

**不要给 AI 太多限制。**

很多人习惯：

- 指定文件路径
- 指定实现方式
- 指定技术栈

这样会限制 AI。

# 第九部分：不只是 Vibe Coding（15 min）

AI 不只是写代码。

它可以用于：

- 设计协作
- 会议准备
- 沟通理解

例如：

开发者用 AI 做交互原型  
再和设计师讨论

这样比：

- 写文档
- 画静态图

更高效。

因为：

**交互本身才是设计。**

# 第十部分：AI 做设计（5 min）

AI 设计的特点：

平均设计

- 什么都有
- 但没有个性

解决方法：

- 使用 design skill
- 提供风格约束
- 多轮迭代

# 第十一部分：使用 AI 后的变化（10 min）

一个明显变化：

**信息获取方式改变**

从构造 query  
到直接表达需求

例如：

过去：

- 搜索多个关键词
- 筛选答案

现在：

- 直接描述问题

整体效率更高。
